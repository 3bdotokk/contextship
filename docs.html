What is ContextShip?

ContextShip is the infrastructure layer between your AI agents and the knowledge they need. It provides deterministic context routing, semantic memory orchestration, and enterprise governance for multi-agent systems.

Modern AI agents fail not because of model limitations, but because of context fragmentation. ContextShip solves this by engineering reliable context pipelines—ensuring every agent receives precisely the right information at the right time.

💡
Key Insight: Context is the new infrastructure. Just as databases solved data persistence and message queues solved async communication, ContextShip solves context delivery for intelligent systems.

2. Initialize the Client

src/context.ts
import { ContextShip } from '@contextship/sdk';
import { SemanticRouter } from '@contextship/router';

const client = new ContextShip({
  apiKey: process.env.CONTEXTSHIP_API_KEY,
  region: 'us-east-1',
  semanticLayer: {
    validation: true,
    deduplication: true,
    compression: 'adaptive'
  }
});

const router = new SemanticRouter(client, {
  strategy: 'deterministic',
  fallback: 'nearest-semantic',
  maxLatency: '10ms'
});

3. Route Your First Context

src/pipeline.ts
// Define a context pipeline
const pipeline = client.createPipeline({
  name: 'finance-agent-context',
  sources: ['market-data', 'user-portfolio', 'risk-models'],
  transforms: [
    { type: 'semantic-filter', threshold: 0.85 },
    { type: 'temporal-window', range: '24h' },
    { type: 'dedup', strategy: 'content-hash' }
  ]
});

// Route context to your agent
const result = await router.route({
  pipeline,
  target: 'finance-ai-agent',
  context: {
    query: 'Analyze portfolio risk for Q4',
    constraints: { maxTokens: 4096, freshness: '1h' }
  }
});

console.log(result.routed);    // true
console.log(result.latency);   // '7ms'
console.log(result.semantic);   // { score: 0.94, validated: true }
✓
That's it! Your agent now receives deterministically routed, semantically validated context. The router automatically handles deduplication, caching, and fallback strategies.

Core Concepts

Architecture

Understanding the layered architecture of ContextShip's semantic context infrastructure.

System Overview

ContextShip operates as a multi-layered system designed to handle the complete lifecycle of context in AI agent systems. Each layer is independently scalable and observable.

L1
Agent Interface Layer SDK, API Gateway, Protocol Handlers
↓
L2
Context Router Deterministic Routing, Semantic Matching, Load Balancing
↓
L3
Semantic Layer Validation, Deduplication, Compression, Coherence
↓
L4
Memory Orchestration Vector Store, Temporal Index, Cross-Agent Sync
↓
L5
Governance Engine Policy Enforcement, Audit, Compliance, Access Control
↓
L6
Execution Systems Agent Runtime, Pipeline Executor, State Manager

Data Flow

Context flows through the system in a deterministic pipeline. Each stage applies transformations, validations, and routing decisions before passing to the next layer.

Architecture Flow
// Simplified context lifecycle
Agent Request → API Gateway → Authentication
    → Context Router (deterministic path selection)
    → Semantic Layer (validate + transform)
    → Memory Orchestration (retrieve + merge)
    → Governance Engine (policy check)
    → Execution (deliver to target agent)
    → Observability (trace + metrics)

Design Principles

  • Deterministic by default: Same input always produces same routing decisions
  • Semantic-first: All operations understand meaning, not just bytes
  • Observable at every layer: Full tracing from ingestion to delivery
  • Governance-native: Policies enforced at the infrastructure level
  • Horizontally scalable: Each layer scales independently
Core Concepts

Context Routing

Deterministic, semantic-aware routing of context to AI agents.

How Routing Works

The Context Router is the core decision engine that determines which context reaches which agent, when, and in what form. Unlike simple retrieval systems, ContextShip routes context based on semantic understanding, agent state, and governance policies.

routing-config.ts
import { Router, Strategy } from '@contextship/router';

const router = new Router({
  strategy: Strategy.DETERMINISTIC,
  
  // Routing rules
  rules: [
    {
      match: { agent: 'finance-*', intent: 'analysis' },
      sources: ['market-data', 'risk-models'],
      transform: 'temporal-filter',
      governance: 'pii-redact'
    },
    {
      match: { agent: 'support-*', intent: 'resolution' },
      sources: ['knowledge-base', 'ticket-history'],
      transform: 'relevance-rank',
      governance: 'customer-data-policy'
    }
  ],

  // Fallback when no rule matches
  fallback: {
    strategy: 'nearest-semantic',
    maxResults: 10,
    minScore: 0.75
  }
});

Routing Strategies

Core Concepts

Semantic Memory

Hierarchical memory architecture for persistent, cross-agent state.

Memory Architecture

ContextShip implements a three-tier memory hierarchy inspired by human cognitive systems:

  • Working Memory: Short-lived, high-bandwidth context for active tasks (TTL: minutes)
  • Episodic Memory: Task-specific memories with temporal indexing (TTL: hours-days)
  • Semantic Memory: Long-term knowledge crystallized from agent interactions (TTL: persistent)
memory-engine.ts
import { MemoryEngine, Tier } from '@contextship/memory';

const memory = new MemoryEngine({
  tiers: {
    working: { capacity: '128MB', ttl: '15m' },
    episodic: { capacity: '2GB', ttl: '7d' },
    semantic: { capacity: 'unlimited', ttl: 'persistent' }
  },
  vectorIndex: {
    dimensions: 1536,
    metric: 'cosine',
    quantization: 'int8'
  },
  sync: {
    mode: 'cross-agent',
    conflictResolution: 'last-write-wins',
    consistency: 'eventual'
  }
});

// Store a memory
await memory.store({
  content: 'User prefers conservative investment strategies',
  tier: Tier.SEMANTIC,
  metadata: { agent: 'finance-ai', confidence: 0.92 }
});

// Retrieve relevant memories
const memories = await memory.recall({
  query: 'What are the user investment preferences?',
  topK: 5,
  minScore: 0.8,
  tiers: [Tier.SEMANTIC, Tier.EPISODIC]
});

Governance

Enterprise-grade policy enforcement, audit trails, and compliance for AI context systems.

Policy Engine

The Governance Engine enforces organizational policies at the infrastructure level. Every context operation passes through policy checkpoints before execution.

governance.ts
import { Governance, Policy } from '@contextship/governance';

const governance = new Governance({
  policies: [
    Policy.piiRedaction({ fields: ['ssn', 'email', 'phone'] }),
    Policy.dataResidency({ region: 'eu-west-1' }),
    Policy.accessControl({ rbac: true, abac: true }),
    Policy.rateLimiting({ maxRequests: 10000, window: '1m' }),
    Policy.contentFiltering({ 
      blocklist: ['competitor-data'],
      sensitivity: 'high'
    })
  ],
  audit: {
    enabled: true,
    retention: '90d',
    destination: 's3://audit-logs/contextship/'
  },
  compliance: ['SOC2', 'GDPR', 'HIPAA']
});

MCP Protocol

Model Context Protocol integration for standardized agent-context communication.

MCP Server Implementation

ContextShip natively implements the Model Context Protocol (MCP), enabling any MCP-compatible client to leverage the full context infrastructure.

mcp-server.ts
import { MCPServer } from '@contextship/mcp';

const server = new MCPServer({
  name: 'contextship-mcp',
  version: '2.4.0',
  capabilities: {
    tools: true,
    resources: true,
    prompts: true
  }
});

// Expose context routing as MCP tool
server.addTool({
  name: 'route_context',
  description: 'Route semantic context to target agents',
  inputSchema: {
    type: 'object',
    properties: {
      query: { type: 'string' },
      target: { type: 'string' },
      constraints: { type: 'object' }
    }
  },
  handler: async (input) => {
    return await router.route(input);
  }
});

// Expose memories as MCP resources
server.addResource({
  uri: 'contextship://memory/{agentId}',
  handler: async (uri) => {
    return await memory.getAgentMemories(uri.params.agentId);
  }
});

server.listen({ transport: 'stdio' });

Orchestration

Complex multi-step workflows with checkpointing, rollback, and deterministic replay.

Pipeline Definition

Orchestration pipelines define the flow of context through multiple processing stages. Each stage can transform, validate, or route context independently.

pipeline.yaml
name: enterprise-context-pipeline
version: "2.0"

stages:
  - name: ingestion
    type: source
    config:
      sources: [knowledge-base, real-time-feeds]
      batch_size: 100
      dedup: true

  - name: semantic-processing
    type: transform
    config:
      embedding_model: contextship-embed-v3
      chunking: semantic
      overlap: 64

  - name: governance-check
    type: policy
    config:
      policies: [pii-redaction, access-control]
      on_violation: quarantine

  - name: routing
    type: router
    config:
      strategy: deterministic
      targets: auto-discover

observability:
  tracing: true
  metrics: true
  alerting:
    latency_p99: 50ms
    error_rate: 0.1%
